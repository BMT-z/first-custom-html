creating/pulling a repo

    Steps

  1. Creation

    2. Step 1 is creating the repo, this is done by going to Github and in the top right there should a plus symbol that you click to create it

    Note: it is best to include a README file when creating this repo

  2. Copying/Cloning

    3. So now our Repo exists its time to get it onto our computer and start coding and adding to it, on the repo page on Github click the green code button and copy the SSH URL

    Note: Very important you click on the SSH option to copy the code as if you do not it won't work properly

  3. Directory

    4. It is recommended to create a directory specifically for repos, to create a repo do the mkdir command, then move into said directory with the command

    Once you have created and moved into te directory you clone your project into here using the following command ( git clone SSH URL )

    Note:The pasted ssh key should look like this git@github.com:USER-NAME/REPOSITORY-NAME.git there should be No https at the beginning, if there is then you've copied the wrong URL

  4. Moving

    5. Now the repo and our local files should be connected, to confirm it connected correctly you can move into the new folder (using cd) that was created then do the command ( git remote -v ) which should display the url to the relevant repo

    I'm assuming the index,html or whatever files were created before the repo linking, so what you should do now is close visual studio code if you have it open to avoid any errors, then find the path to the repo file using cd and the target files using a combination of cd and ls, once you find the targe path do the ( mv ) command then target the file you want to move followed by the target destination, example ( mv Downloads/test1.txt repos/tests )

  5. Changes

    6. Any changes made in this directory will need to be pushed for it to show on your repo, changes include any modifications even deletion of a file or renaming

    If you did move some files to this new repo directory you will already have a change ready to be staged, if you didn't you can just create a text file using the touch command

    Now once you've made a change if you do the command ( git status ) it should show your current file in red which indicates it's not staged, to stage any changes made you do ( git add NameOfFile.txt ) if you do the git status command again it should now show your file/s in green indicating they're ready for a commit

    Note: if you want to add multiple files to the staging area you do ( git add . ) and this adds all files to the staging area

  6. Committing

    7. The next stage after you've added your changes to the staging area is to commit these changes, the purpose of a commit is to make it easier to navigate the versions of your files and see what changes were made (especially useful when working with others) so its always best these contain a lot of accurate information

    To do a commit you do the commands ( git commit -m "message describing what you've done" ), if you want a real life example, before I push these changes to my repo my commit command is going to look like this $ git commit -m "expanded step 5: changes, added step 6: committing, typo fixes"

    Note: sometimes you might forgot what you've added, deleted or changed so what I like to do is have a txt document opened while coding and update it as I go along so the commit message is very accurate, also make sure your actual commit message is within speech marks " "

  7. Pushing (p)

    8. At this point all you code is staged, committed and finally ready to be pushed, which is the term git uses to 'push' a file change live to the repo in this case

    Just to make do a final check do git status and it should show your latest commit comment, if it does then do ( git push ) and that should be it

    Note: to check all pushes and commit you can do the command ( git log ) and this should show that information in reverse chronological order as well as who made the commit and when, if you have a lot of commits and pushes you can use the arrow keys to go down the list if not everything shown and to exit the log you simply press the letter "q" on your keyboard
